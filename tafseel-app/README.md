## منصة تفصيل – الرحلات الأساسية

هذا المستودع يحتوي على نموذج أولي تفاعلي مبني بإطار Next.js يغطّي الرحلات الأربع الرئيسة في منصة **تفصيل**:

- رحلة الزائر الجديد من الصفحة الرئيسية وحتى اختيار الدور.
- رحلة العميل من لوحة المول، شحن المحفظة (BNPL 70/30)، وحتى إتمام الشراء.
- رحلة المصمم من رفع التصميم (Auto-Mint NFT) ووضعه في المتجر.
- رحلة المصنع من إدارة الكتالوج إلى تنفيذ الطلب وتأكيد الشحن.
- رحلة المسوّق من إنشاء رابط تتبع (Affiliate) وحتى الحصول على QR مخصص.

جميع الشاشات والتدفقات مكتوبة بالعربية وباتجاه RTL مع حالة محفظة مشتركة عبر الصفحات (Context) لإظهار أثر الشحن والشراء.

## التشغيل محلياً

1. تثبيت الاعتمادات:

   ```bash
   npm install
   ```

2. تشغيل خادم التطوير:

   ```bash
   npm run dev
   ```

3. افتح المتصفح على العنوان [http://localhost:3000](http://localhost:3000).

## خريطة الصفحات المهمة

| المسار | الوصف |
| --- | --- |
| `/` | الصفحة الرئيسية – عرض ركائز المنصة وروابط التسجيل. |
| `/register` | نموذج إنشاء الحساب. |
| `/onboarding/role` | اختيار الدور (عميل، مصمم، مصنع، مسوّق). |
| `/app/mall` | لوحة العميل – المول (افتراضي). |
| `/app/wallet` | محفظة تفصيل (شحن BNPL، توزيع 70/30، وبعد الشراء). |
| `/app/product/[id]` | صفحة منتج في المول مع تفاصيل السعر. |
| `/app/product/[id]/checkout` | شاشة تأكيد الدفع مع خصم الرصيد المقفل أولاً. |
| `/designer/dashboard` | لوحة المصمم مع الإحصائيات والتصاميم. |
| `/designer/upload` → `/designer/dual-choice` → `/designer/store-setup` → `/designer/success` | رحلة المصمم الكاملة من التوثيق وحتى نشر المنتج. |
| `/factory/dashboard` | لوحة المصنع (الطلبات). |
| `/factory/catalog` → `/factory/catalog/new` | إدارة الكتالوج وإضافة منتج خام. |
| `/factory/orders/[id]` | تنفيذ الطلب (تحميل ملف التصميم، تأكيد الشحن). |
| `/marketer/dashboard` | لوحة المسوّق – أداء الحملات. |
| `/marketer/campaigns/new` → `/marketer/campaigns/success` | إنشاء رابط تتبع وحصول على QR. |

## ملاحظات تقنية

- المشروع يستخدم **App Router** مع مجلد `src/app`.
- تم استخدام خط Google **Cairo** لدعم اللغة العربية.
- تمت إضافة **WalletProvider** لمشاركة حالة المحفظة بين صفحات رحلة العميل.
- تم بناء المكونات باستخدام Tailwind CSS (إصدار v4) مع تخصيصات بسيطة في `globals.css`.

لأي تحديثات مستقبلية، يمكن إضافة منطق حقيقي للربط مع واجهات برمجية أو قواعد بيانات مكان العناصر التجريبية الحالية.
